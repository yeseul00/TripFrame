# Expert 3: 제품/UX 전문가 리뷰

> **문서 ID**: TF-REVIEW-003
> **작성일**: 2026-03-29
> **리뷰어**: 모바일 앱 제품 기획/UX 디자이너
> **대상**: TripFrame Phase 1~3 완료, Phase 4 계획 단계

---

## 1. Executive Summary

> **핵심 결론**: TripFrame은 "핵심 가치는 명확하지만, 포장이 덜 된" 상태입니다. 역산 엔진의 6단계 투명성과 공백 감지의 즉각적 시각 신호는 앱의 최고 강점이지만, 온보딩 전무 + 설정 미적용 + 예약 후 추적 부재가 출시 전 반드시 해결해야 할 과제입니다.

## 2. 기능 완결성

### 2.1 약속 vs 동작

| 기능 | 사용자 기대 | 실제 동작 | 갭 |
|------|-----------|---------|-----|
| 역산 계산 | 집 출발 시간 자동 계산 | ✅ 정확 동작 (09:15) | 없음 |
| 공백 감지 | 이동수단 누락 구간 발견 | ✅ DANGER 2건 감지 | 여유시간 미표시 |
| 교통수단 제안 | 옵션 비교 + 예약 링크 | ✅ 4개 옵션 + 가격 | 예약 후 추적 없음 |
| 대안 교통수단 | 다른 수단 시간 변화 | ✅ Δ시간 배지 | 없음 |
| 여행 관리 | CRUD | ✅ 전부 동작 | 없음 |

| 설정 | 짐/교통/여유도 | ❌ **UI만, 저장·적용 안 됨** | **심각** |
| 클라우드 동기화 | 다기기 접근 | ❌ **OAuth 400 오류** | **심각** |

## 3. 사용자 여정 분석

### 여정 1: 새 여행 만들기 — 4/5

- ⚠️ 날짜 입력이 텍스트 필드 (Phase 4 DatePicker로 전환 예정)

### 여정 2: 이동수단 누락 찾기 — 3.5/5

- ✅ DANGER 카드 빨간 시각 신호 즉각적
- ✅ 구체적 구간 메시지 ("하카타에서 유후인")
- ✅ 요약 카드 (위험 공백/총 공백/완성 구간)
- ⚠️ "공백감지" 메뉴명 직관성 부족
- ⚠️ 여유시간(Free Time) 미표시

### 여정 3: 교통수단 비교 + 예약 — 3/5

- ✅ OptionCard 정보 밀도 적절 (요금/시간/모드)
- ✅ 추천 옵션 퍼플 테두리 강조
- ✅ 인원수 반영 합산 비용 실시간 업데이트
- ❌ **예약 후 앱 이탈 → "해결됨" 표시 없음 → 루프 끊김**

### 여정 4: 출발 시각 역산 — 4.5/5 (최고점)

- ✅ Constitution VI-4 준수: 6단계 역산 근거 투명
- ✅ 대안 교통수단 Δ시간 배지 직관적
- ✅ Day 선택 + 자유시간 카드 연동
- ⚠️ 설정 연동 미완 (Phase 4에서 해결 예정)

### 여정 5: 첫 사용 경험 — 1/5 (치명적)

- ❌ **온보딩 플로우 전무**
- ❌ 앱 첫 진입 시 Mock 데이터가 표시 → "이 데이터가 뭐지?" 혼란
- ❌ 핵심 가치("여행 일정의 빈 칸을 찾아주는 앱") 3초 안에 전달 불가

## 4. 정보 구조(IA) 분석

### 현재: 5탭 구조

일정 / 공백감지 / 제안카드 / 역산 / 설정

### 발견된 문제

**문제 1: 공백감지 ↔ 제안카드 역할 중복**
- 공백감지 탭: GapCard 펼치면 옵션 표시
- 제안카드 탭: Gap별 옵션 표시
- 사용자 관점: "문제 발견 → 해결책"이 연속 동작인데 두 탭으로 분리
- 제안: 통합하여 "이동 체크" 탭으로 만들고, 빈 슬롯에 D-day/체크리스트 배치

**문제 2: 설정 탭이 메인 탭에 위치**
- 설정은 자주 접근하지 않는 기능
- 프로필 아이콘 또는 홈 화면 설정 메뉴로 이동 검토

**문제 3: 탭 간 크로스 네비게이션**
- 화면설계서에 배지 탭 → 관련 탭 이동 설계
- 실제 동작 여부 사용자 테스트에서 명시적 확인 필요

## 5. 디자인 시스템 평가

### 강점

- 색상 의미 체계 명확: DANGER(빨강), WARNING(노랑), SUCCESS(초록), Accent(보라)
- 타이포그래피 계층 정의: 20px → 16px → 14px → 13px → 11px
- 다크 테마 기본: 어두운 환경(비행기, 호텔)에서 가독성 우수
- 간격 체계(Spacing tokens): xs(4) → sm(8) → md(12) → lg(16) → xl(24) → xxl(32)

### 약점

- 라이트 테마 미지원 → 밝은 야외 가독성 저하 가능
- 이벤트 유형 아이콘이 이모지 의존 → 플랫폼 간 렌더링 차이 가능
- 접근성(a11y): VoiceOver/TalkBack 라벨 전수 검토 미시행 (IDEA-030)

## 6. 피드백 반영도 평가

### Phase 3 피드백 → 즉시 수정 (★★★★★)

| 피드백 | 수정 내용 | 결과 |
|--------|---------|------|
| BUG-001: 홈 버튼 겹침 | 공통 헤더 바 추가 | ✅ |
| BUG-002: 빈 일정 화면 없음 | Empty state + "첫 이벤트 추가" 버튼 | ✅ |
| BUG-003: 빈 여행 이벤트 저장 안 됨 | Day 1 타임라인 자동 생성 | ✅ |
| BUG-004: 역산 탭 Mock 데이터 잔류 | Empty state 분기 처리 | ✅ |

피드백 수집 → 즉시 코드 수정 → E2E 재검증의 반복 속도가 매우 우수합니다. Phase 3에서 사용자 테스트 시나리오 문서(5개 태스크, 관찰 체크리스트)까지 준비한 것은 체계적인 UX 프로세스를 보여줍니다.

## 7. 출시 준비도

### Ready to ship (6개)
- 역산 + 공백감지 핵심 가치
- Trip/Event CRUD
- Transport comparison
- Alternative transport
- Offline persistence

### Must fix before launch (3개)
- 설정 실기능화 → Phase 4
- 날짜 Picker → Phase 4
- 메뉴명 변경 → Phase 4

### Critical gaps (6개)
- 온보딩 플로우 (zero guidance)
- Google OAuth 수정 (cloud sync blocked)
- 로컬 데이터 암호화 (legal risk)
- Development Build (never tested on device)
- App Store 메타데이터 (screenshots, description, privacy policy)
- Crash reporting / analytics (no Sentry)

**판정**: Phase 4 완료 후에도 앱 스토어 출시까지 1~2 Phase 추가 필요

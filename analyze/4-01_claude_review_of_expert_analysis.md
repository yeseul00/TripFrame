## 1. 전체 평가

리뷰 구조 자체는 완성도가 높다. 3명 독립 리뷰 → 공통 교집합 추출 → 통합 보고서로 연결되는 흐름이 논리적이고, 기술 부채 ID 체계(TD-01~09, UX-01~04, BIZ-01~02)와 Phase 연계가 실제 작업 계획에 바로 쓸 수 있는 수준이다.

---

## 2. 동의하는 핵심 발견

### 공통 3대 과제 — 정확하게 짚었다

| 과제 | 판단 | 근거 |
|------|------|------|
| TD-02 설정 미적용 | Critical 맞음 | `useSettingsStore`가 존재하지만 `calculateReverseTime` 등 Core 함수에 전혀 전달되지 않는 상태 |
| TD-04 AsyncStorage 평문 | Critical 맞음 | 개인정보보호법 제29조 리스크 지적은 실제 앱 출시 시 법적 문제 소지 |
| UX-01 온보딩 전무 | P0 맞음 | Mock 데이터 첫 노출 문제는 Phase 3 사용자 테스트에서도 실제로 확인된 이슈 |

**역산 엔진 6단계 투명성 → 최고 차별점** 판단에도 동의한다. Expert 3이 4.5/5.0 준 유일한 항목이며, 실제로 앱의 핵심 가치 명제다.

---

## 3. 이의 또는 보완이 필요한 부분

### 3.1 D-day 위젯을 "사업 생존 요건"으로 분류한 것 (Expert 1) — 과도한 우선순위

현 단계에서 과도하게 높은 우선순위다. D-day 위젯은 WidgetKit + Development Build 전환이 선행되어야 하고, 설정 실기능화·OAuth 수정·보안 강화보다 먼저 해야 할 이유가 없다. "중요하지만 Phase 5 이후가 맞는 기능"이다. Expert 1의 "사업 생존 요건" 표현은 로드맵 압박을 줄 수 있어 주의가 필요하다.

**수정 권고**: D-day 위젯 → Phase 5로 이동. Phase 4에서는 설계 문서만 작성.

### 3.2 TD-03 Development Build Critical 분류 — 시점 재검토 여지

"Critical(프로덕션 차단)"으로 분류되어 있지만, 현재 Expo Go로 Web E2E 86건 100% 통과한 상황에서 Phase 4 전체를 블로킹하는 요건으로 두기는 과도할 수 있다. Phase 4에서 네이티브 기능(expo-secure-store 등)을 실기기 검증할 때 필요하므로, **Phase 4 중반**으로 재조정이 현실적이다.

**수정 권고**: TD-03 심각도 Critical → High 재분류. Phase 4 후반~Phase 5 초반 타이밍.

### 3.3 "공백감지 + 제안카드 탭 통합" 권고 (Expert 3) — 구현 복잡도 누락

탭 통합은 UX 관점의 이점이 명확하지만, 현재 `GapDetectionScreen`과 `SuggestionsScreen`의 상태 구조가 분리되어 있어 통합 시 Store 재설계가 필요하다. 리뷰에서 이 구현 난이도가 낮게 표시되지 않았지만, Phase 5에서도 상당한 리팩토링 비용이 있다는 점이 누락되어 있다.

**추가 명기 필요**: 탭 통합 = `useGapStore` 신규 설계 + 두 화면 병합 작업 포함. 난이도 "상"으로 재분류 권고.

### 3.4 Expert 1 보험 제안 카드 "Phase 4에 UI만이라도" 제안 — 역효과 우려

제안 자체는 좋으나, 보험사 제휴 없이 UI만 먼저 배치하면 "클릭해도 아무것도 안 되는" 버튼이 된다. Expert 3이 설정 미적용을 "고장난 기능 인상"으로 지적한 것과 정확히 같은 문제를 만들 수 있다.

**수정 권고**: Phase 4에서는 설계 문서만 작성. 보험 UI는 제휴 확정 이후 배치.

### 3.5 FreeTimeResult 타입 위치 이슈 (TD-07) — 경미하게 과장됨

Constitution III-3 위반이라고 했지만, `FreeTimeResult`는 `freeTime.ts` 함수의 반환 타입이고 다른 모듈에서 직접 임포트하는 경우가 있을 때만 실제 문제가 된다. Phase 4에서 함께 옮기는 건 좋지만 심각도 재분류가 적절하다.

**수정 권고**: TD-07 심각도 Medium → Low 재분류. 기회 있을 때 처리.

## 5. 결론

리뷰의 **사실 진단은 정확**하다. 다만 일부 우선순위 분류(TD-03 Critical, D-day "사업 생존 요건", 보험 UI Phase 4 삽입)는 Phase 4를 과부하시킬 수 있는 요소다. P0/P1 집중 → Phase 4 마감 후 P2/보류 항목을 Phase 5로 넘기는 전략이 현실적이다.

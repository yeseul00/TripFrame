## 1. 문서 개요

### 1.1 목적

TripFrame의 모든 기능/비기능 요구사항을 ID 체계로 관리하고, 화면설계서(TF-SDD-001) · 시스템설계서(TF-SAD-001) · 태스크(tasks.md) · 테스트 시나리오(TF-TC-001)와의 양방향 추적성을 보장한다.

### 1.2 참조 문서

| 문서명 | 문서 ID | 역할 |
|--------|---------|------|
| Constitution | spec-kit/constitution.md | 불변 원칙, 우선순위 (Article IX) |
| 기능 명세서 | spec-kit/spec.md | US-01~06, FR-01~04, NFR, 샘플 데이터 |
| 태스크 명세서 | spec-kit/tasks.md | TASK-001~027, Phase 분류 |
| 앱 화면설계서 | TF-SDD-001 | 화면 ID (SCR-001~004) |
| 시스템 설계서 | TF-SAD-001 | 아키텍처, 엔진 설계, 데이터 모델 |
| 구현 가이드 | spec-kit/implement.md | 검증 시나리오 1~4, 엣지 케이스 |
| E2E 테스트 케이스 | TF-TC-001 | TC-001~027 상세 시나리오 |

### 1.3 요구사항 ID 체계

| 접두사 | 범주 | 번호 범위 |
|--------|------|-----------|
| `REQ-FR-` | 기능 요구사항 (Functional) | 001~099 |
| `REQ-NFR-` | 비기능 요구사항 (Non-Functional) | 001~099 |

각 요구사항에는 다음 속성이 부여된다.

| 속성 | 값 범위 | 설명 |
|------|---------|------|
| 우선순위 | P1 / P2 / P3 / P4 | Constitution Article IX 기준 |
| 난이도 | 상 / 중 / 하 | 구현 난이도 |
| 관련 화면 | SCR-001~004 | 화면설계서 화면 ID |
| 관련 태스크 | TASK-001~027 | tasks.md 태스크 ID |
| 테스트 시나리오 | TC-xxx | 검증 테스트 케이스 ID |

## 3. 비기능 요구사항 (NFR)

### 3.1 오프라인 동작

| ID | 요구사항 | 측정 방법 | 수용 기준 | 출처 |
|----|----------|-----------|-----------|------|
| REQ-NFR-001 | 역산 계산 및 공백 감지는 네트워크 연결 없이 완전 동작한다 | 비행기 모드에서 시나리오 1~4 실행 | 4개 시나리오 모두 Pass | Constitution III-5 |

## 1. 문서 개요

### 1.1 목적

TripFrame의 모든 기능/비기능 요구사항을 ID 체계로 관리하고, 화면설계서(TF-SDD-001) · 시스템설계서(TF-SAD-001) · 테스트 시나리오(TF-TC-001)와의 양방향 추적성을 보장한다.

### 1.2 참조 문서

| 문서명 | 문서 ID | 역할 |
|--------|---------|------|
| Constitution | spec-kit/constitution.md | 불변 원칙, 우선순위 (Article IX) |
| 기능 명세서 | spec-kit/spec.md | US-01~06, FR-01~04, NFR, 샘플 데이터 |
| 태스크 명세서 | spec-kit/tasks.md | TASK-001~027, Phase 분류 |
| 앱 화면설계서 | TF-SDD-001 | 화면 ID (SCR-000~003) |
| 시스템 설계서 | TF-SAD-001 | 아키텍처, 엔진 설계, 데이터 모델 |
| 구현 가이드 | spec-kit/implement.md | 검증 시나리오 1~4, 엣지 케이스 |
| E2E 테스트 케이스 | TF-TC-001 | TC-001~027 상세 시나리오 |

### 1.3 요구사항 ID 체계

| 접두사 | 범주 | 번호 범위 |
|--------|------|-----------|
| `REQ-FR-` | 기능 요구사항 (Functional) | 001~099 |
| `REQ-NFR-` | 비기능 요구사항 (Non-Functional) | 001~099 |

각 요구사항에는 다음 속성이 부여된다.

| 속성 | 값 범위 | 설명 |
|------|---------|------|
| 우선순위 | P1 / P2 / P3 / P4 | Constitution Article IX 기준 |
| 난이도 | 상 / 중 / 하 | 구현 난이도 |
| 관련 화면 | SCR-000~003 | 화면설계서 화면 ID |
| 관련 태스크 | TASK-001~027 | tasks.md 태스크 ID |
| 테스트 시나리오 | TC-xxx | 검증 테스트 케이스 ID |

## 3. 비기능 요구사항 (NFR)

### 3.1 UI/UX 원칙

| ID | 요구사항 | 수용 기준 | 
|---|---|---|
| REQ-NFR-001 | 바텀 시트 패럴랙스 | 바텀 시트(일정)가 위로 확장될 때, 배경 지도 영역이 scale 축소 및 opacity 감소하며 패럴랙스 효과를 낸다. | 
| REQ-NFR-002 | 무베젤(Bezel-less) 디자인 | 브라우저 기본 스크롤바를 완전히 제거하고, 드래그 핸들의 시각적 요소를 최소화하여 투명한 터치 영역으로 UX를 보장한다. | 
| REQ-NFR-003 | 라이트/다크 모드 지원 | 사용자의 기기 설정에 따르거나 수동 토글로 다크모드와 라이트모드 전환을 지원해야 한다. | 
| REQ-NFR-004 | 시각적 피로도 최소화 | '미예약' 등의 상태는 크리티컬 에러(Red)가 아닌 Action Required(Orange) 컬러를 사용하여 사용자 불안감을 낮춘다. |

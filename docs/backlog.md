# TripFrame Backlog

> 이 문서는 TripFrame 프로젝트의 아이디어 수집·평가·승격 퍼널이다.
아이디어가 떠오르면 **Inbox에 한 줄 추가**하면 된다. 형식 부담 없음.
Phase 마감 시점에 리뷰하여 Evaluated → Committed 또는 Dropped로 이동한다.
> 

**리뷰 주기**: Phase 마감 시점 (또는 아이디어 10개 누적 시)
**리뷰 기준**: Constitution Article IX 정합성, 기존 요구사항 관계, 난이도/영향 범위
**승격 경로**: Inbox → Evaluated → Committed → 요구사항 정의서(REQ-FR-030~) 또는 시스템 설계서 개정

## 운영 가이드

### 아이디어 추가 방법

1. Inbox 테이블 맨 아래에 행 추가
2. ID는 `IDEA-NNN` 순번 부여
3. 날짜, 아이디어 한 줄, 카테고리 이모지, 출처, 메모 기입
4. **형식 부담 없이** 한 줄이면 충분

### 리뷰 프로세스

1. **트리거**: Phase 마감 또는 Inbox 10개 누적
2. **체크 항목**:
    - Constitution Article IX 우선순위 위반 여부
    - 기존 REQ-FR/NFR과의 관계 (신규 vs 확장)
    - 대략적 난이도/영향 범위
    - 선행 조건 충족 여부
3. **판정**: 승격(Evaluated) 또는 보류(Dropped)
4. **승격 시**: 정식 문서 반영
    - 요구사항 → TF-RDS-001 REQ-FR-030~
    - 아키텍처 → TF-SAD-001 12장 개정
    - 구현 → tasks.md TASK-028~

### ID 관리

- IDEA 번호는 삭제/재사용하지 않음
- 승격된 IDEA는 Inbox에서 Evaluated로 이동 (삭제 아님)
- Dropped도 영구 보존 (재검토 가능)

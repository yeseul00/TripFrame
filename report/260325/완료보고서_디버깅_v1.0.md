## 완료된 작업

### Phase 0 — 백업 생성
- 수정 대상 6개 파일을 `tripframe/backup/260325/`에 원본 보존

### Phase 1 — @tripframe/core 모듈 연결
- `apps/mobile/package.json`에 `"@tripframe/core": "workspace:*"` 추가
- **실제 적용**: Windows 한글 경로 제한으로 pnpm 심볼릭 링크 생성 불가 → Metro `extraNodeModules`로 대체
  - `metro.config.js`에 `resolver.extraNodeModules['@tripframe/core']` 직접 경로 명시
  - 이 방식이 Expo 모노레포에서 더 안정적 (pnpm 심볼릭 링크 우회)

### Phase 2 — 빌드 설정 수정
- `babel.config.js`: NativeWind v4에서 제거된 `"nativewind/babel"` preset 삭제
- `metro.config.js`: `disableHierarchicalLookup: true` 제거 + `extraNodeModules` 추가

### Phase 3 — 타입 정합성
- `packages/core/src/types/trip.ts`: `Gap.severity` 타입을 spec 기준으로 변경
  - `'low' | 'medium' | 'high'` → `'DANGER' | 'WARNING' | 'OK'`
  - `GapSeverity` export 타입 추가
- `packages/core/src/data/mock.ts`: 기존 `'high'` 값을 `'DANGER'`로 전체 교체

### Phase 4 — GapEngine 로직 보완
- `gapEngine.ts` 전면 개선:
  - 시간 기반 severity 3단계 판정 구현 (spec FR-02 충족)
  - `DANGER`: 이동 수단 없음
  - `WARNING`: 이동 수단 있으나 여유 < 30분
  - `OK`: 여유 ≥ 30분 (공백으로 보고하지 않음)
  - `diffMinutes()` 헬퍼 함수로 HH:mm 시간 차이 계산
- engine.test.ts에 `severity: 'DANGER'` 검증 케이스 추가

### Phase 5 — 헌법 위반 수정
- `useTripEngine.ts`: `reverseCalcData` 반환값 추가
- `ReverseCalcDetailScreen.tsx`: `@tripframe/core` 직접 import 제거 → hook 경유

### 추가 수정 (테스트 환경 안정화)
- `jest.setup.js`: `jest.mock`을 setupFiles에서 실행해 발생하는 오류 해결 (내용 제거)
- `jest.config.js`:
  - `lucide-react-native` → `__mocks__/lucide-react-native.js` 매핑 추가
- `__mocks__/lucide-react-native.js` 신규 생성: 아이콘 모듈 테스트 mock

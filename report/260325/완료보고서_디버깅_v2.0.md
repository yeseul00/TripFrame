## 4. 핵심 기술 결정 사항 (다음 세션 인계)

### (1) @tripframe/core 모듈 연결 방식
**결정**: pnpm 심볼릭 링크 대신 Metro `extraNodeModules` 직접 매핑
**이유**: Windows + 한글 경로에서 pnpm이 EISDIR 오류로 심볼릭 링크 생성 실패
**위치**: `apps/mobile/metro.config.js`
```js
config.resolver.extraNodeModules = {
  '@tripframe/core': path.resolve(workspaceRoot, 'packages/core/src'),
};
```
**영향**: `packages/core`를 빌드(`pnpm build`)하지 않아도 `src/`를 직접 참조하므로, 코어 수정 후 빌드 불필요. jest도 동일하게 `src/` 직접 참조.

### (2) lucide-react-native jest mock
**결정**: `__mocks__/lucide-react-native.js` 파일 생성 + `jest.config.js` moduleNameMapper 등록
**이유**: 루트 `node_modules`에만 설치된 패키지를 한글 경로 때문에 jest가 resolve 못함
**위치**: `apps/mobile/__mocks__/lucide-react-native.js`

### (3) GapEngine severity 기준
**결정**: spec FR-02 기준 3단계 판정
- `DANGER`: 이동 수단 없음 (위치가 다른데 transport 타입 이벤트 없음)
- `WARNING`: 이동 수단 있으나 시간 여유 < 30분
- `OK`: 정상 (Gap으로 보고하지 않음)

## 6. 잔여 문제 및 다음 작업

### [즉시 필요] Metro 번들링 검증
```bash
cd tripframe/apps/mobile
pnpm start    # 또는 node ".../pnpm.cjs" start
```
에뮬레이터 또는 실기기로 다음 화면 동작 확인:
- `일정` 탭: Day 1/2/3 타임라인 렌더링
- `공백감지` 탭: Day 2, Day 3 DANGER 공백 카드 표시
- `역산` 탭: 09:15 출발 시간 및 단계별 계산 표시

### [블로커 가능성] NativeWind v4 peer dependency 불일치
```
WARN: nativewind@4.2.3 → tailwindcss@~3 필요
현재 설치: tailwindcss@4.2.2
```
**예상 증상**: Metro 번들링은 성공하나 스타일(색상, 레이아웃)이 적용 안 될 수 있음
**해결 방법 후보**:
- A) `tailwindcss`를 v3으로 다운그레이드 (`pnpm add tailwindcss@3`)
- B) `nativewind`를 v4 호환 최신 버전으로 업그레이드
- C) Metro 실행 후 실제 문제 발생 시 대응 (현재 영향 범위 불확실)

## 2. 작업 Phase

### Phase 0 — 백업 생성
**목적**: 수정 전 모든 대상 파일의 원본 보존
**대상 파일**:
- `apps/mobile/package.json`
- `apps/mobile/babel.config.js`
- `apps/mobile/metro.config.js`
- `apps/mobile/src/screens/ReverseCalcDetailScreen.tsx`
- `packages/core/src/types/trip.ts`
- `packages/core/src/logic/gapEngine.ts`

**완료 기준**: `backup/260325/` 하위에 6개 파일 저장 확인

---

### Phase 1 — 모노레포 의존성 연결 (P1 해결)
**목적**: `@tripframe/core`를 `apps/mobile`의 공식 의존성으로 등록해 pnpm 심볼릭 링크 생성

**작업**:
1. `apps/mobile/package.json` `dependencies`에 추가:
   ```json
   "@tripframe/core": "workspace:*"
   ```
2. `tripframe/` 루트에서 `pnpm install` 실행
3. `node_modules/@tripframe/core` 심볼릭 링크 생성 확인

**검증**: `ls tripframe/apps/mobile/node_modules/@tripframe/core`

---

### Phase 2 — NativeWind / Metro 설정 수정 (P2, P3 해결)
**목적**: NativeWind v4 호환 babel 설정 + Metro 안정화

**작업**:
1. `babel.config.js` — `"nativewind/babel"` preset 줄 제거
2. `metro.config.js` — `disableHierarchicalLookup: true` 줄 제거

**검증**: `pnpm start` 실행 후 번들링 오류 없음 확인

---

### Phase 3 — 타입 정의 spec 정합성 맞추기 (P4 해결)
**목적**: `Gap.severity` 타입을 헌법/spec 기준으로 통일

**작업**:
1. `packages/core/src/types/trip.ts`
   - 변경 전: `severity: 'low' | 'medium' | 'high'`
   - 변경 후: `severity: 'DANGER' | 'WARNING' | 'OK'`
2. `packages/core/src/data/mock.ts` — 기존 mock 데이터의 severity 값 갱신
   - `'high'` → `'DANGER'`
3. `packages/core/dist/` 재빌드: `pnpm build` (packages/core)

**검증**: `pnpm typecheck` 오류 없음

---

### Phase 4 — GapEngine 로직 보완 (P5 해결)
**목적**: spec FR-02 기준 severity 3단계 판정 구현

**현재 로직 한계**:
- 위치 문자열 단순 비교만 수행
- 시간 기반 판정(30분 미만 → WARNING) 없음
- 항상 `'high'` severity 고정

**수정 후 로직**:
```
이동수단 타입 이벤트 없는 구간 → DANGER
이동수단 있으나 시간 여유 < 30분 → WARNING
이동수단 있고 시간 여유 ≥ 30분 → OK
```

**검증**: `pnpm test` (packages/core) — engine.test.ts 전체 통과

---

### Phase 5 — ReverseCalcDetailScreen 헌법 위반 수정 (P6 해결)
**목적**: screen이 core를 직접 import하는 헌법 위반 제거

**작업**:
1. `useTripEngine` hook에 `reverseCalcData` 반환값 추가
2. `ReverseCalcDetailScreen.tsx`의 `import ... from '@tripframe/core'` 제거
3. hook에서 데이터 수신하도록 변경

**검증**: `pnpm test` (apps/mobile) — integration.test.tsx 통과

---

### Phase 6 — 최종 통합 검증
**목적**: 전체 수정 후 앱 실행 및 테스트 일괄 확인

**작업**:
1. `packages/core` 빌드: `pnpm build`
2. core 테스트: `pnpm test` (packages/core)
3. mobile 테스트: `pnpm test` (apps/mobile)
4. 앱 기동: `pnpm start` (apps/mobile)
5. 3개 탭(일정/공백감지/역산) 동작 육안 확인

## 사용 피드백 요약

### [FB-001] 일정 선택 홈 화면 필요 — 우선순위: 높음

**내용**: 첫 화면에 개인별 홈(일정 선택 페이지) 추가 필요. 여행이 여러 개인 경우를 대비해 시간 역순으로 노출, 신규 일정 카드가 가장 상단에 위치해야 함.

**현재 상태**: 앱 진입 시 단일 Mock 여행의 타임라인이 바로 노출됨.

**영향도**: 실사용 시나리오에서 필수 — 여행 1개 이상 관리 불가.

**매핑 태스크**: TASK-057

---

### [FB-002] 제안카드 탭 — Gap 알림 토글 인터랙션 — 우선순위: 중간

**내용**: 각 Gap 알림 카드를 탭했을 때 해당 알림에 대한 선택 사항(OptionCard 목록)이 토글(accordion) 방식으로 열리고 닫혀야 함. 현재는 항상 펼쳐진 상태.

**현재 상태**: SuggestionScreen에 모든 옵션이 펼쳐진 상태로 고정 노출.

**영향도**: 여러 Gap이 있을 때 스크롤 부담 증가.

**매핑 태스크**: TASK-058

---

### [FB-003] 역산 탭 — DAY 선택 및 자유 시간 표시 — 우선순위: 중간

**내용**:
1. 역산 화면에 Day 선택 아이콘/탭이 없음.
2. DAY 별 공백(자유) 시간 계산 결과가 노출되어야 함.

**현재 상태**: 역산 탭은 단일 앵커 이벤트 기준 고정 계산만 표시. Day 선택 없음.

**영향도**: 여행 중 여유 시간 파악 불가.

**매핑 태스크**: TASK-059

---

### [FB-004] 역산 탭 — 대안 교통수단 기회비용 토글 — 우선순위: 중간

**내용**: 현재 버스로 계산된 역산 결과에서, 철도 등 다른 교통수단 선택 시의 출발 시각 변화(기회비용)를 토글 방식으로 노출. 일정 변경 시 참고 목적.

**현재 상태**: 고정된 단일 경로 기준 역산만 표시.

**매핑 태스크**: TASK-060

---

### [FB-005] 설정 탭 — 내용 긍정 평가, 추가 항목 검토 필요 — 우선순위: 낮음

**내용**: 현재 설정 탭 구성(짐 크기/교통 선호/여유도/피드백)은 잘 구성됨. 추가 설정 항목 개발 고민 필요.

**현재 상태**: 3가지 선호도 + 피드백 버튼. Google 로그인 연동.

**후보 항목 (미확정)**: 언어 설정, 통화 단위, 알림 설정, 기본 인원수, 자주 쓰는 출발지 등.

**매핑 태스크**: TASK-061

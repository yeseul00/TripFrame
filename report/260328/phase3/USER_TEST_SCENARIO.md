# 사용자 테스트 시나리오 스크립트

**버전**: 1.0
**작성일**: 2026-03-28
**대상**: 여행 계획 경험이 있는 성인 (20~40대)
**소요 시간**: 약 20~30분

---

## 테스트 환경 안내

> 진행자가 테스터에게 전달할 내용:
>
> "지금 보여드릴 앱은 여행 일정에서 이동 수단이 빠진 구간을 자동으로 찾아주는 앱입니다.
> 정해진 정답은 없으며, 자연스럽게 사용해 주시면 됩니다.
> 사용하시면서 궁금하거나 불편한 점이 있으면 말씀해 주세요.
> 앱이 아닌 분이 평가받는 것이니 편하게 해주세요."

**접속 방법**: `http://[테스트 서버 IP]:8082` 또는 QR 코드 (Expo Go)

---

## 태스크 1: 새 여행 만들기

**목표**: 홈 화면에서 새 여행을 생성할 수 있다.

**지시문**:
> "오사카 여행을 계획 중입니다. 앱에 새 여행을 추가해 보세요."

**관찰 포인트**:
- [ ] "+ 새 여행 만들기" 버튼을 찾는 데 걸리는 시간
- [ ] 여행명 필드가 필수임을 인지하는가
- [ ] 저장 후 카드가 목록에 추가됨을 확인하는가
- [ ] 모달 헤더 "새 여행"을 인지하는가

**완료 기준**: 홈 목록에 새 여행 카드 표시

---

## 태스크 2: 여행 일정 확인

**목표**: 기존 여행(후쿠오카 · 유후인)을 선택해 일정을 탐색한다.

**지시문**:
> "후쿠오카 여행 일정이 이미 있습니다. 이 여행을 열고 어떤 일정이 있는지 확인해 보세요."

**관찰 포인트**:
- [ ] 여행 카드 탭하면 탭 화면으로 진입함을 이해하는가
- [ ] 일정 탭에서 Day 탭 전환을 시도하는가
- [ ] 이벤트 항목(✈️, 🏨 등)을 이해하는가

**완료 기준**: 일정 탭에서 최소 1개 Day의 이벤트 확인

---

## 태스크 3: 이동 수단 누락 확인

**목표**: 공백감지 탭에서 이동 수단이 없는 구간을 파악한다.

**지시문**:
> "이 여행 일정에서 이동 수단이 빠진 구간이 있다고 합니다. 어떤 구간인지 찾아보세요."

**관찰 포인트**:
- [ ] "공백감지" 탭을 자연스럽게 탭하는가
- [ ] DANGER 카드를 인지하고 탭해서 펼치는가
- [ ] "하카타 → 유후인" 구간 메시지를 읽는가
- [ ] 위험/경고 구분을 이해하는가

**완료 기준**: 이동 수단 없는 구간 2개 식별

---

## 태스크 4: 이동 수단 제안 확인

**목표**: 제안카드 탭에서 교통수단 옵션과 비용을 확인한다.

**지시문**:
> "하카타에서 유후인으로 이동하는 방법을 찾아보세요. 2명이 갈 예정입니다."

**관찰 포인트**:
- [ ] "제안카드" 탭을 찾는가
- [ ] 인원 선택(1→2명)을 시도하는가
- [ ] 고속버스/열차/택시 옵션을 스크롤하며 비교하는가
- [ ] 비용 정보를 읽고 선택 의도를 보이는가

**완료 기준**: 2명 기준 총 비용 확인

---

## 태스크 5: 출발 시각 역산 확인

**목표**: 역산 탭에서 공항 출발을 위해 집에서 몇 시에 나가야 하는지 확인한다.

**지시문**:
> "귀국하는 날, 공항에 2시간 전 도착하려면 집에서 몇 시에 나가야 하는지 확인해 보세요."

**관찰 포인트**:
- [ ] "역산" 탭을 탭하는가
- [ ] 역산 결과 카드(출발 시각)를 찾는가
- [ ] "다른 교통수단으로 계산" 토글을 탭하는가
- [ ] 대안 교통수단과 Δ시간 배지를 이해하는가

**완료 기준**: 역산 출발 시각 읽기

## 테스트 빌드 접속 방법

### 웹 (PC/Mobile 브라우저)
```
http://localhost:8082
```
> 진행자 PC에서 `python3 -m http.server 8082 --directory dist` 실행 후 공유

### Expo Go (실제 기기)
```bash
cd tripframe/apps/mobile
npx expo start --tunnel
# QR 코드 스캔 (Expo Go 앱 필요)
```

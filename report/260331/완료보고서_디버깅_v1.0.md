### Phase 1 — suggestion.spec.ts beforeEach 수정

**원인**: `MoveCheckScreen` 진입 시 첫 DANGER Gap이 자동으로 펼쳐진 상태. `beforeEach`에서 해당 카드를 한 번 더 클릭 → 닫힘 → 교통 옵션 사라짐 → SCR-004-02~08 전부 실패.

**수정 내용** (`e2e/suggestion.spec.ts:19`):
```ts
// 변경 전 — 이미 열린 카드를 닫아버림
await page.getByText('DANGER').first().locator('..').locator('..').click();

// 변경 후 — 자동 펼침 상태 확인만
await page.waitForSelector('text=DANGER', { state: 'visible', timeout: 5000 });
```

**해결**: SCR-004-02~08 7건 통과

### Phase 2 — gap.spec.ts / moveCheck.spec.ts TC-014 계열 수정

**원인**: 테스트가 "닫힌 카드 → 클릭 → 열림" 흐름을 가정했으나, 카드가 이미 열린 상태 → 클릭 → 닫힘 → 실패.

**수정 방침**:
- **토글 동작 검증** (TC-014): 닫기 → 열기 순서로 명시적 검증
- **펼쳐진 상태 확인** (TC-014-B~D, US-002-01): 클릭 제거, 바로 `toBeVisible()` 확인
- **예약 완료 클릭** (US-002-02): 카드 클릭 없이 바로 버튼 클릭
- **교통 필터 테스트** (SCR-004-09, 10, 07): 설정 변경 후 이동 체크 진입 시 자동 펼침 활용

**해결**: TC-014 계열 16건 통과

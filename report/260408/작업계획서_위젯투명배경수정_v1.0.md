# TripFrame 위젯 투명 배경 수정 작업계획서

**작성일**: 2026-04-08
**작업 유형**: Android 위젯 렌더링 버그 수정
**관련 태스크**: TASK-102 (D-day 위젯 POC)
**목표**: 홈화면 위젯이 투명하게 보이는 문제 해결 → 배경색 `#0F0F13` 정상 렌더링

---

## 1. 현상

- Android 홈화면에 TripFrame 위젯 추가 시 배경이 투명하게 보임
- 위젯 내용(텍스트)도 표시되지 않거나 흐릿하게 보임
- `TripWidgetProvider.tsx` 에 `backgroundColor: '#0F0F13'` 이 설정되어 있음에도 적용 안 됨

---

## 2. 근본 원인 분석

### 원인 A — `rn_widget.xml` (initialLayout) 미존재 [심각도: 높음]

`widgetprovider_tripwidget.xml`이 `@layout/rn_widget`을 `initialLayout`으로 참조하고 있으나,
해당 파일(`android/app/src/main/res/layout/rn_widget.xml`)이 **존재하지 않음**.

```xml
<!-- widgetprovider_tripwidget.xml -->
android:initialLayout="@layout/rn_widget"   ← 이 파일이 없음
```

Android는 JS 렌더링 전에 `initialLayout`을 먼저 보여주는데, 파일이 없으면
위젯 영역이 빈 상태(투명)로 표시됨. 빌드는 성공하나 런타임에 fallback 없이 투명 처리됨.

### 원인 B — JS backgroundColor의 RemoteViews 한계 [심각도: 중간]

`react-native-android-widget`은 React Native 컴포넌트를 Android `RemoteViews`로 변환함.
JS에서 `backgroundColor: '#0F0F13'`을 설정해도, 일부 Android 버전/런처에서
RemoteViews의 background 속성으로 변환이 불완전하게 동작하는 경우 있음.
→ 네이티브 드로어블(ShapeDrawable)로 명시하는 것이 더 안정적.

### 원인 C — Android 12+ 위젯 요구사항 미충족 [심각도: 중간]

`widgetprovider_tripwidget.xml`에 Android 12+(API 31+) 필수/권장 속성 누락:

| 속성 | 현재 | 필요 |
|------|------|------|
| `android:widgetFeatures` | 없음 | `reconfigurable\|configuration_optional` |
| `android:previewLayout` | 없음 | `@layout/widget_preview` (권장) |
| `android:targetCellWidth/Height` | 없음 | 2×1 기준 권장 |

Android 12에서 위젯 배경 처리 방식이 바뀌어, 명시적 background drawable 없으면 시스템이 투명 처리할 수 있음.

## 3. 해결 방안

### Phase 1 — `rn_widget.xml` 생성 (필수)

**파일 생성**: `android/app/src/main/res/layout/rn_widget.xml`

```xml
<?xml version="1.0" encoding="utf-8"?>
<FrameLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    android:background="@drawable/widget_background">
</FrameLayout>
```

역할: JS 렌더링 전 초기 상태로 단색 배경을 보여줌.

---

### Phase 2 — 네이티브 배경 드로어블 생성 (필수)

**파일 생성**: `android/app/src/main/res/drawable/widget_background.xml`

```xml
<?xml version="1.0" encoding="utf-8"?>
<shape xmlns:android="http://schemas.android.com/apk/res/android"
    android:shape="rectangle">
    <solid android:color="#0F0F13" />
    <corners android:radius="16dp" />
</shape>
```

`TripWidgetProvider.tsx`의 `FlexWidget`에도 이 드로어블을 참조하도록:

```tsx
<FlexWidget
  style={{
    height: 'match_parent',
    width: 'match_parent',
    justifyContent: 'center',
    alignItems: 'center',
    backgroundColor: '#0F0F13',
    // react-native-android-widget v0.20+ 지원 시
    // backgroundDrawable: 'widget_background',
  }}
>
```

> `backgroundDrawable` 속성 지원 여부는 라이브러리 버전에 따라 다름.
> JS `backgroundColor`와 네이티브 드로어블 둘 다 설정하는 것이 가장 안전.

---

### Phase 3 — `widgetprovider_tripwidget.xml` Android 12+ 대응

**파일 수정**: `android/app/src/main/res/xml/widgetprovider_tripwidget.xml`

```xml
<?xml version="1.0" encoding="utf-8"?>
<appwidget-provider xmlns:android="http://schemas.android.com/apk/res/android"
    android:minWidth="180dp"
    android:minHeight="110dp"
    android:targetCellWidth="2"
    android:targetCellHeight="1"
    android:resizeMode="none"
    android:description="@string/widget_tripwidget_description"
    android:initialLayout="@layout/rn_widget"
    android:previewLayout="@layout/rn_widget"
    android:updatePeriodMillis="0"
    android:widgetFeatures="reconfigurable|configuration_optional"
    android:widgetCategory="home_screen">
</appwidget-provider>
```

변경 항목:
- `android:targetCellWidth/Height` 추가 (Android 12+ 그리드 힌트)
- `android:previewLayout` 추가 (위젯 선택 화면 미리보기)
- `android:widgetFeatures` 추가 (Android 12+ 필수)

## 4. 작업 순서

```
1. widget_background.xml 드로어블 생성
2. rn_widget.xml 레이아웃 생성 (widget_background 참조)
3. widgetprovider_tripwidget.xml 수정 (Android 12+ 속성 추가)
4. npx expo run:android 으로 네이티브 재빌드
5. 에뮬레이터/기기에서 위젯 추가 → 배경색 확인
6. 라이트/다크 모드 전환 후 재확인
```

> **주의**: `android/` 디렉터리 수정 후에는 **반드시 네이티브 재빌드** 필요.
> `expo start`(Metro only)로는 네이티브 변경사항이 반영되지 않음.

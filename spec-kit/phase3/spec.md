# Feature Specification: TripFrame Phase 3

**Feature**: `003-tripframe-phase3`
**Branch**: `003-tripframe-phase3`
**Status**: Planning
**Created**: 2026-03-28
**Depends On**: `002-tripframe-phase2` (Phase 2 complete ✅)

---

## 1. Overview

### Phase 2 완료 현황

Phase 2는 2026-03-28 웹 기준 완료됐습니다 (E2E 60/60 통과). 현재 제한사항:

- 앱 진입 시 단일 Mock 여행만 표시 — 여행 목록/선택 화면 없음
- 여행 및 이벤트 생성/수정/삭제 불가 (CRUD 없음)
- 제안카드: 모든 Gap의 옵션이 항상 펼쳐진 상태 — 토글 없음
- 역산 탭: Day 선택 없음, 자유 시간(여유 시간) 미표시
- 역산 탭: 단일 교통수단 기준 고정 계산 — 대안 비교 없음

### Problem Statement

Phase 2가 "어떤 교통수단이 나에게 맞나?"를 답했다면, Phase 3는:

- "내 여행이 여러 개인데 앱에서 선택할 수 없어" → 홈/여행 목록 화면 없음
- "새 여행을 만들거나 일정을 수정하고 싶어" → Trip/Event CRUD 없음
- "제안카드에 옵션이 너무 많이 펼쳐져 있어 불편해" → 토글 인터랙션 없음
- "역산에서 다른 날 계획도 보고 싶고, 여유 시간이 얼마나 되는지 알고 싶어" → Day별 자유시간 없음
- "버스 대신 기차를 탔을 때 몇 시에 출발해야 하는지 비교하고 싶어" → 대안 교통수단 비교 없음

### Solution

Phase 3는 **여행 데이터 관리 + UX 정교화**를 중심으로 합니다:

1. **홈 화면**: 여행 목록 → 여행 선택 → 타임라인 진입
2. **Trip CRUD**: 여행 생성/수정/삭제
3. **Event CRUD**: 이벤트 추가/수정/삭제
4. **제안카드 토글**: Gap별 옵션 accordion 방식
5. **역산 개선**: Day 선택 + 자유 시간 + 대안 교통수단 비교

## 2. User Stories

### US-P3-001: 홈 화면 — 여행 목록 관리

**As a** 여행자
**I want to** 앱 진입 시 내 여행 목록을 보고 원하는 여행을 선택하거나 새 여행을 만들고 싶다
**So that** 여러 여행 계획을 동시에 관리할 수 있다

**Acceptance Criteria**:
- [ ] 앱 진입 시 여행 목록 화면(홈)이 표시된다
- [ ] 여행 목록은 출발일 역순(최신 우선)으로 정렬된다
- [ ] "새 여행 만들기" 카드가 가장 상단에 표시된다
- [ ] 여행 카드에는 여행명, 목적지, 기간이 표시된다
- [ ] 여행 카드를 탭하면 해당 여행의 타임라인 화면으로 진입한다

---

### US-P3-002: Trip CRUD

**As a** 여행자
**I want to** 새 여행을 만들고, 기존 여행을 수정하거나 삭제하고 싶다
**So that** 실제 여행 계획을 앱에서 직접 관리할 수 있다

**Acceptance Criteria**:
- [ ] 새 여행 생성: 여행명, 목적지, 출발일·귀국일 입력 가능
- [ ] 기존 여행 수정: 여행 정보 편집 가능
- [ ] 여행 삭제: 확인 모달 후 삭제
- [ ] 로그인 시 Supabase에 자동 동기화

---

### US-P3-003: Event CRUD

**As a** 여행자
**I want to** 여행에 이벤트(항공편, 호텔, 이동 등)를 추가/수정/삭제하고 싶다
**So that** 실제 일정으로 공백 감지와 역산을 활용할 수 있다

**Acceptance Criteria**:
- [ ] 이벤트 추가: 날짜, 시간, 장소, 이벤트 유형 입력
- [ ] 이벤트 수정: 기존 이벤트 편집
- [ ] 이벤트 삭제: 이벤트 제거
- [ ] 이벤트 저장 후 Gap 감지 자동 재계산

---

### US-P3-004: 제안카드 토글 인터랙션

**As a** 여행자
**I want to** 제안카드에서 각 Gap 알림을 탭하면 해당 구간의 교통 옵션만 펼쳐지고 닫히길 원한다
**So that** 여러 Gap이 있을 때 필요한 구간만 집중해서 볼 수 있다

**Acceptance Criteria**:
- [ ] 기본 상태: 각 Gap 헤더만 표시 (옵션 숨김)
- [ ] Gap 헤더 탭 → 해당 Gap의 OptionCard 펼쳐짐
- [ ] 펼쳐진 Gap 헤더 재탭 → 닫힘
- [ ] 여러 Gap 동시에 펼침 가능 (독립 토글)

---

### US-P3-005: 역산 탭 Day 선택 + 자유 시간

**As a** 여행자
**I want to** 역산 탭에서 날짜를 선택하고, 해당 날의 자유 시간을 확인하고 싶다
**So that** Day별 출발 시각과 여유 시간을 파악할 수 있다

**Acceptance Criteria**:
- [ ] 역산 탭 상단에 Day 선택 탭/아이콘 표시
- [ ] Day 선택 시 해당 Day의 역산 결과 표시
- [ ] `calculateFreeTime()` 결과 (도착 후 체크인 전 자유 시간) 표시
- [ ] 자유 시간 활용 제안 (식사, 관광 등) 노출

---

### US-P3-006: 역산 탭 대안 교통수단 비교

**As a** 여행자
**I want to** 현재 역산 기준 교통수단 외에 다른 교통수단을 선택했을 때의 출발 시각도 보고 싶다
**So that** 교통수단 변경 시 일정에 미치는 영향을 파악할 수 있다

**Acceptance Criteria**:
- [ ] 역산 결과 하단에 "다른 교통수단으로 계산" 토글 표시
- [ ] 토글 시 현재 구간의 대안 교통수단 목록 표시
- [ ] 각 대안 선택 시 변경된 출발 시각 즉시 계산·표시
- [ ] 기준 계산과 대안 계산의 시각 차이(Δ분) 표시

## 3. Out of Scope (Phase 4)

- 예약 알림 (Booking alerts)
- 패스 경제성 분석 (Pass economics)
- 활동 추천 (Activity recommendations)
- Apple OAuth
- 푸시 알림

---

## 4. Non-Functional Requirements

- Trip/Event CRUD는 오프라인 우선 (로컬 저장 후 온라인 시 동기화)
- 새 화면 추가 시 NativeWind 다크 테마 일관성 유지
- 신규 비즈니스 로직은 `packages/core`에 순수 함수로 구현
- 단위 테스트 커버리지 ≥80% 유지
